# SEO Playbook

Use this playbook when a page or query is already showing some traction and you
need to decide what to do next.

The default goal is not "publish more content."

The default goal is to:

- find where demand is already showing up
- diagnose the real bottleneck
- choose the right page action
- improve the page in a way that matches search intent

## Default workflow

Start from the previous day's Google Search Console data.

Look at:

- impressions
- queries
- pages already getting traction

Then diagnose the main issue first.

In most cases the problem is one of these:

- low CTR
- low position

That diagnosis decides whether you should:

- improve an existing page
- create a new page

After that, validate in Semrush:

- check search volume
- find stronger commercial or high-intent variations
- review the top-ranking pages that actually match the product

If the top 10 results are mostly tutorial-style blogs, that usually means blog
intent.

In that case:

- create a similar blog if the topic deserves its own page
- also consider inserting part of that content into an existing page that
  already has impressions

That insert is usually best as:

- a supporting section
- a use-case section
- a case-study-style section

## If CTR is low

The usual reasons are:

- the meta title is not compelling enough
- the meta description is not compelling enough
- the page content is not rich enough yet

Default actions:

- rewrite the title and description so the promise is clearer
- make the page feel more complete and more credible
- add richer sections if the page still feels thin

## If position is low

The usual actions are:

- assess whether a related supporting blog is needed
- add more relevant high-volume keywords naturally into the page
- benchmark higher-ranking pages and strengthen the page with better sections,
  more interactive content, and a stronger conversion CTA

## How to choose keywords

Start from three sources:

- recent queries from the site itself, especially target variations and keyword
  clusters
- competitor pages, to spot new keyword clusters tied to the business
- PostHog data, to understand what users are actually searching for or landing
  on

Competitor sites like `Artisan`, `Outscraper`, and `D7 Leads` are useful for
finding new keyword directions.

Also check Semrush every `1-2` weeks to see whether stronger competitors are
emerging around core terms like:

- `local business database`
- `google maps scraper`

When a strong page shows up:

- add it to a competitor watch list
- use it for reverse analysis

PostHog helps in two ways:

- recordings sometimes reveal the exact search terms users mention
- entry URLs reveal which feature or use case pulled them in

One important limitation:

Without new product features, keyword expansion gets harder.

`local business` may remain the core, while many higher-volume adjacent
keywords are not actually ICP-fit.

There are always long-tail opportunities, but large-scale expansion there is
usually a programmatic SEO problem, not a one-page copywriting problem.

## How to judge keyword quality

Good keywords usually meet all three checks:

- search volume is at least `150+`
- most of the top 10 results are relevant competitors or category-related pages
- there are at least `3+` related SEO or GEO question queries showing real
  demand

A keyword can have volume and still be weak.

Common failure cases:

- the traffic is too broad
- the traffic is too international
- the intent is not close to buying intent
- there is no real question-based demand behind it

## How to structure pages and posts

Beyond meta tags and image alts, the page structure should include:

- a clear `H1` containing the main keyword
- `H2`s covering the most important indexable topics
- useful sections, not filler
- internal links inside relevant sections
- a related-content block
- FAQ
- testimonials or proof
- a bottom CTA aligned with the page intent

Each section should do one of three jobs:

- help the page rank
- qualify the traffic
- move the user deeper into the site

## Internal linking rules

- relevance matters most
- use a breadcrumb path at the top:
  - `Home > Category > Current page`
- add contextual internal links inside content sections
- add a related-content block for further exploration
- use CTA links that direct traffic toward product or conversion pages

## Refresh rules

- update pages with fresher data and references so the content clearly looks
  current
- consolidate multiple weak low-impression pages into one stronger page when
  overlap is high
- refresh social proof

## What metrics to watch

- clicks
- position
- impressions by query
- CTR
- top-performing feature pages and data pages

What those usually signal:

- clicks help show which non-branded traffic is actually entering the site
- position matters because moving into the top 3 often changes traffic
  dramatically
- impressions by query reveal demand and new content opportunities
- CTR usually points to title and meta-description issues
- top-performing feature or data pages show what use cases are actually pulling
  people in

PostHog should be used after the click:

- understand where users drop off
- check whether the page promise matches the product reality

If users arrive for one feature description and later discover the product is
not exactly that, the page copy usually needs adjustment.

## Default page outline

1. Hero section with the core keyword in the `H1`
2. Pain point or use-case section with long-tail keywords
3. Core content sections broken into `H2`s and `H3`s
4. Internal-linking section
5. FAQ
6. Bottom CTA

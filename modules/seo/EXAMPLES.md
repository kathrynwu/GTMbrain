# SEO Examples

These are pattern examples, not universal answers.

Use them to shape the decision, the page brief, and the next action.

## Example 1: Existing page has impressions but low CTR

Input:

- page: product page about `google maps scraper`
- impressions: `1,800`
- CTR: `0.8%`
- average position: `5.9`

Interpretation:

- the page is already visible enough
- position is not the main problem
- the snippet promise is weak or the page looks too thin

Recommended motion:

`RECOMMENDATION: Choose A because the page already has traction and the fastest lift is improving the existing page before creating anything new.`

- `A` improve the existing page now
- `B` create a supporting page later if blog intent is also showing up
- `C` hold if the page is attracting the wrong ICP entirely

What to change first:

- rewrite the title to make the commercial use case clearer
- rewrite the meta description so the value prop is obvious
- strengthen the page with richer sections:
  - use cases
  - proof
  - FAQ
  - stronger CTA

Example direction:

- title angle: `Google Maps Scraper for Local Business Leads | {brand}`
- description angle: focus on finding, exporting, and qualifying local business
  data instead of generic scraping language

## Example 2: Existing page has low position

Input:

- page: category page targeting `local business database`
- impressions: `900`
- CTR: `1.9%`
- average position: `13.4`

Interpretation:

- CTR is acceptable for the current rank
- the main issue is that the page is not strong enough yet

Recommended motion:

`RECOMMENDATION: Choose A because the keyword is already attached to an existing page, but the page needs stronger sections and broader topical coverage.`

- `A` improve the existing page
- `B` create one supporting blog if there is a clear subtopic gap
- `C` stop if the keyword is not ICP-fit

What to change first:

- benchmark top-ranking category pages
- add more natural coverage of relevant high-volume variations
- add stronger sections:
  - who the data is for
  - how the dataset is used
  - how fresh the data is
  - proof and testimonials
  - clearer bottom CTA

## Example 3: Query has blog intent

Input:

- query: `how to build a local business prospect list`
- current ranking pages: mostly tutorials and blog posts
- product page already has a few impressions on related terms

Interpretation:

- the SERP is telling you this is mostly informational intent
- forcing a product page to rank alone is usually the wrong move

Recommended motion:

`RECOMMENDATION: Choose B because the SERP is blog-heavy, so you should create a supporting post and also use part of that content to strengthen the existing product page.`

- `A` push only the product page
- `B` create a supporting blog and feed the strongest parts back into the product page
- `C` hold if the topic is high-volume but not ICP-fit

What to do:

- write the blog around the exact educational intent
- reuse part of that material inside the product page as:
  - a process section
  - a use-case section
  - a case-study-style section

## Example 4: Keyword quality check

Input keyword:

- `local business database`

Quality check:

- search volume: above `150+`
- top 10 results: relevant competitors and category pages
- question demand: multiple related SEO and GEO queries exist

Interpretation:

- this is a good core keyword candidate

Counter-example:

- a broad adjacent keyword with volume but weak question demand
- top results are mixed, international, or not product-relevant

Interpretation:

- volume alone does not make it a good keyword

## Example 5: Sample page brief

Target:

- main keyword: `google maps scraper`
- page type: product page
- goal: attract high-intent users looking for local-business data extraction

Suggested structure:

1. Hero with the core keyword in the `H1`
2. Pain point and use-case section using long-tail variations
3. Core content sections:
   - what the tool does
   - what data users can export
   - who uses it
   - how it compares to manual workflows
4. Internal links to:
   - prospecting
   - enrichment
   - outbound follow-through
5. FAQ
6. Bottom CTA aligned with product intent

What this page should do:

- rank for the category term
- qualify whether the visitor wants leads, enrichment, or workflow help
- move the user into the right product or playbook next step

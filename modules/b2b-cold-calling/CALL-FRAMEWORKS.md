# B2B Cold Call Frameworks

Use these as framework shapes, not scripts to repeat word for word.

The rule:

- mention the problem as the hook
- then move into the value prop

## 1. Initial cold call, first touch

### Prep, `60s`

Before dialing:

- check news if relevant
- check LinkedIn
- check Crunchbase if relevant
- open the CRM page

### Pattern-interrupt opener

Ask for a short window immediately.

Example:

`Hi {{Name}}, it's {{Rep}} at {{Company}}, can I steal 30 seconds to explain why I'm calling?`

### Persona hook

State the peer challenge.

Example:

`I talk to RevOps leads drowning in bounced emails and stale contacts.`

### Value statement

Deliver the outcome metric.

Example:

`We push phone-verified contacts plus intent scores into Salesforce, cutting list-build time by 10 hours a week.`

### Discovery question

Probe current state.

Example:

`How are you handling contact accuracy today?`

### Handle objection

Use the objection library in [`OBJECTION-LIBRARY.md`](./OBJECTION-LIBRARY.md).

### Meeting CTA

Offer two precise time slots.

Example:

`Worth a 15-minute compare? I'm free Tuesday at 10:00 or Wednesday at 3:00 PT.`

### Confirm and close

Restate the next step and send the invite.

Example:

`Great, sending the calendar invite now. Appreciate the chat.`

## 2. Follow-up call, touch 2-N

Follow-ups must add new information each time, or they sound like spam.

Use this shape:

1. reference context
2. add new value
3. re-ask for the meeting

Example:

1. `Following up on the voicemail about your SDR list-build time.`
2. `Pulled a free 200-contact sample in your exact ICP.`
3. `Ten minutes tomorrow to review it?`

## 3. Gatekeeper route

If you hit a gatekeeper:

- ask confidently for the target by name
- if questioned, give brief value for the target's department
- politely ask for the best time or voicemail if blocked

Rule:

- respect beats deception
